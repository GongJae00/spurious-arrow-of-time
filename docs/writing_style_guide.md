# Writing Style Guide

This guide prevents the manuscript from reading like a generic generated
explanation. Use it during drafting and revision.

## Target Style

The paper should sound like:

```text
formal
compact
technical
evidence-constrained
clear about scope
```

The paper should not sound like:

```text
popular science
philosophy
corporate project documentation
grant rhetoric
defensive rebuttal
AI-generated overview
```

## Sentence Rules

Preferred:

```text
short declarative sentences
active voice when clear
specific technical nouns
one claim per sentence where possible
```

Avoid:

```text
long multi-clause sentences
empty intensifiers
grand framing without evidence
repeating the same claim across sections
```

Review every sentence longer than 25-30 words.

## Paragraph Rules

Every paragraph must have one role:

```text
motivate a problem
define a mechanism
describe a control
report evidence
state a boundary
```

Reject a paragraph if it:

```text
mixes motivation, methods, and results
repeats an earlier paragraph
contains a claim not in docs/claim_evidence_matrix.md
requires a reader to infer the experiment design from prose alone
```

## Claim Wording Ladder

Use the strongest supported wording and no stronger.

Allowed for the main phenomenon:

```text
We show that sequence ERM can rely on a non-causal irreversible nuisance arrow
in a controlled benchmark.
```

Allowed with boundary:

```text
Counterfactual nuisance replacement improves mean OOD accuracy but fails the
seed-stability gate.
```

Use carefully:

```text
robust
causal
irreversibility
arrow of time
demonstrate
```

Forbidden unless evidence changes:

```text
solves
universal
physical entropy production
heat dissipation
thermodynamic causality
real-world deployment
```

## Bad vs Better

Bad:

```text
Time is a fundamental and fascinating phenomenon, and neural networks may learn
the wrong direction of time in a way that reveals deep truths about intelligence.
```

Better:

```text
In inverse tasks generated by irreversible dynamics, the useful temporal
asymmetry is the one that identifies the hidden cause. A separate irreversible
nuisance can create a stronger but non-causal shortcut.
```

Bad:

```text
This paper proposes a powerful new benchmark and method for solving selective
irreversibility.
```

Better:

```text
The benchmark isolates a failure mode in which a sequence predictor succeeds
IID but fails when a train-correlated nuisance arrow reverses OOD.
```

Bad:

```text
Our method is robust because it achieves much better OOD performance.
```

Better:

```text
Counterfactual replacement improves mean OOD accuracy from 0.124 to 0.756, but
passes the OOD threshold in only 7/10 seeds.
```

## Section-Specific Tone

Introduction:

```text
Precise motivation. No philosophy. No result dump.
```

Related Work:

```text
Contrast research lines. Do not write a bibliography in prose.
```

Methods/Benchmark:

```text
Concrete definitions. Every control should have a reason.
```

Results:

```text
Numbers first, interpretation second. No hiding failed gates.
```

Discussion:

```text
Strong on phenomenon, bounded on method, honest on synthetic assumptions.
```

Abstract:

```text
Write last. No citations. No unbounded claims.
```

## Forbidden Habits

Do not:

```text
start many paragraphs with "This paper"
use "novel" as a substitute for explaining the gap
write "we believe" for evidence-backed statements
write "we demonstrate" for claims not passing gates
bury limitations in vague future-work language
use "significant" unless statistical significance is actually tested
```

Clarification:

```text
The phrase "this paper" is not banned. Generic, detached, boilerplate prose is
the problem.
```

## Revision Checklist

For every paragraph:

```text
[ ] One point only.
[ ] Evidence exists if a result is claimed.
[ ] No unsupported adjectives.
[ ] No repeated motivation.
[ ] No hidden overclaim.
[ ] No sentence over 30 words unless mathematically necessary.
[ ] Reads like a journal article.
```

For every section:

```text
[ ] Section role is clear.
[ ] First paragraph tells the reader why the section exists.
[ ] Last paragraph transitions cleanly.
[ ] Figures/tables are referenced where they answer the claim.
```

## English Editing Target

Before submission, revise as if applying a strict technical editing pass:

```text
remove filler
reduce repeated phrases
replace vague nouns with specific terms
keep tense consistent
define abbreviations once
avoid mixing American and British spelling
check grammar and punctuation mechanically
```
